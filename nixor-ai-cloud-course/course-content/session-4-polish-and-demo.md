# Session 4 — Polish, Explain, and Demo Like an Engineer

**Goal:** ship a polished version, understand real-world constraints, and present your work clearly.

## Big ideas for today
- **Engineering value = code + communication**.
- **Cost and performance are design constraints**, not afterthoughts.
- **Responsible AI** means handling uncertainty and misuse consciously.

## Steps
1. **Polish UX**: improve readability, wording, and flow so a first-time user can succeed quickly.
2. **Polish reliability**: handle one failure scenario gracefully (bad input, empty prompt, API failure).
3. **Review cloud cost drivers**: identify what increases spend in your app and what controls it.
4. **Explain your architecture**: app runtime, model endpoint, secrets/config, deployment target.
5. **Prepare a 3-minute demo**: problem, approach, live walkthrough, and one lesson learned.
6. **Run final rehearsal**: do one uninterrupted end-to-end run from login to live app usage.
7. **Present and reflect**: what would you build next if you had 2 more weeks?

## Starter track
- Deliver a clean demo with one solid use case and one clear takeaway.

## Stretch track
- Show comparative prompting (before/after) and justify design decisions with evidence.

## You learned
How to ship, evaluate, and communicate an AI cloud product in a way that maps to real industry practice.
